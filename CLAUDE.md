## Critical SDK & Pattern Gotchas

These are the most important gotchas â€” memorize these to avoid common bugs:

- **`plugin.switchTo()` needs the INTERNAL prefixed name** (e.g. `ThcContact`), NOT `publicName` (`Contact`). The `publicName` is for UI labels and TypeScript type names only.
- **Always `.pipe(window.toMainInstance(true))`** on all `fetchAllRecords()` / `fetchOneRecord()` queries
- **SDK records have NON-ENUMERABLE properties** â€” `{ ...record }` and `Object.keys(record)` produce `{}` / `[]`. Use `record.getState()` to get a plain JS object with all properties.
- **Records are immutable** â€” never `Object.assign(record, data)` or `record.field = value`, always spread `{ ...existingData, ...newData }`
- **Subscription payloads are PLAIN objects** (no `getState()`), do NOT include `id`, and have `null`/`undefined` for unchanged fields â€” never blind-merge, only merge defined non-null values, and always preserve the known `id`
- **Mutations disrupt subscriptions** â€” after `mutation.execute()`, clean up and re-subscribe
- **Direct GraphQL `fetch()` is an alternative to SDK for complex queries** â€” useful for calc/aggregation queries, cross-model joins, `orderBy`, and `field()` aliasing. The SDK query builder *does* support calc queries; past failures were due to invalid field references, not SDK limitations.
- **Keep `.limit()` reasonable** â€” limits above ~1,000 can cause the SDK to hang. Use pagination instead.
- **Vite dev server**: root: `.` with `open: '/dev/'` â€” NOT `root: 'dev'` which breaks `../src/` paths
